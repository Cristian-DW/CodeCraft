1. ¿Qué es HTML?

HTML (HyperText Markup Language) es el esqueleto de cualquier página web.

Imagina HTML como los cimientos, vigas y paredes de una casa. No define el color de la pintura (eso es CSS) ni cómo funciona la puerta del garaje (eso es JS), solo define qué es cada cosa.

La versión actual es HTML5, que introdujo nuevas capacidades semánticas y multimedia.

2. Anatomía de una Etiqueta

HTML se basa en etiquetas (tags). 

<p class="texto-grande"> Hola, soy un párrafo </p>
  • Apertura: <p> (indica dónde empieza el elemento).
  • Atributo: class="..." (da información extra, como estilos).
  • Contenido: "Hola, soy un párrafo".
  • Cierre: </p> (indica dónde termina, ¡nota la barra inclinada!).

3. Títulos y Párrafos

Los encabezados (Headings) dan jerarquía. 

<h1>El título más importante (Solo 1 por página)</h1>
<h2>Subtítulo</h2>
<h3>Sub-subtítulo</h3>
<p>Esto es un párrafo normal de texto.</p>
Tip Pro: No uses h2 o h3 solo porque quieres letras grandes. Úsalos para estructurar el contenido. Los motores de búsqueda (Google) aman una buena estructura.

4. Listas

Hay dos tipos principales de listas:

1. Desordenada `<ul>`: Puntos (bullets).
2. Ordenada `<ol>`: Números (1, 2, 3...).

Dentro de ambas, cada ítem es un `<li>` (List Item).

5. Enlaces (Links/Anchors)

El hipervínculo es la esencia de la "Web". 

<a href="https://google.com" target="_blank">Ir a Google</a>
- href: Hacia dónde va el link (Hypertext REFerence).
- target="_blank": Abre el link en una pestaña nueva.

6. Imágenes

Una imagen vale más que mil palabras. 

<img src="gato.jpg" alt="Foto de un gato tierno" />
Ojo:
  • Es una etiqueta de cierre automático (no tiene </img>).
  • src: La ruta (URL) de la imagen.
  • alt: Texto alternativo. Si la imagen falla o el usuario es ciego y usa un lector de pantalla, leerá este texto. ¡Es obligatorio para accesibilidad!

Evaluación 2: Multimedia

¿Por qué es importante el atributo 'alt' en las imágenes?

Para que la imagen cargue más rápido.
Para accesibilidad y SEO (Buscadores).
Para ponerle borde a la imagen.

7. HTML Semántico

Antes, todo se hacía con <div> (cajas genéricas).
HTML5 trajo etiquetas con significado para ayudar a leer el código:

  • <header>: Cabecera del sitio.
  • <nav>: Menú de navegación.
  • <main>: Contenido principal.
  • <article>: Un post o noticia independiente.
  • <footer>: Pie de página.

8. Formularios

La forma en que el usuario nos envía datos. 

<form>
  <label>Email:</label>
  <input type="email" placeholder="tu@correo.com" required>
  <button type="submit">Enviar</button>
</form>
El atributo type el input cambia todo: puede ser texto, password, fecha, color, archivo, etc.

9. Tablas

Úsalas PARA DATOS, no para maquetar el sitio. 

<table>
  <tr> <!-- Table Row (Fila) -->
    <th>Nombre</th> <!-- Table Header -->
    <th>Edad</th>
  </tr>
  <tr>
    <td>Ana</td> <!-- Table Data -->
    <td>25</td>
  </tr>
</table>

10. Meta Tags (Head)

Información sobre tu documento que el usuario no ve, pero el navegador sí. 

<head>
  <meta charset="UTF-8"> (Para tildes y ñ)
  <meta name="description" content="Mi web increíble">
  <meta name="viewport" content="..."> (Crucial para celulares)
</head>
Sin el meta viewport, tu web se verá diminuta en un iPhone.

11. Buenas Prácticas

Escribir código limpio es profesional:

  • Indentación: Usa Tab o espacios para anidar elementos. Facilita la lectura.
  • Minúsculas: Usa siempre minúsculas para las etiquetas (`<DIV>` funciona pero se ve horrible).
  • Comentarios: `<!-- Esto es un comentario -->` para dejar notas a tu yo del futuro.
