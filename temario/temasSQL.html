1. ¿Qué es una Base de Datos?

Imagina una hoja de cálculo de Excel gigante y superpoderosa.

Una Base de Datos Relacional (RDBMS) guarda información en Tablas que tienen filas (registros) y columnas (campos).

Ejemplos populares: PostgreSQL (la más avanzada), MySQL (la más popular), SQLite (ligera).

2. Sintaxis Básica

SQL (Structured Query Language) es el inglés de los datos.
Se acostumbra escribir los comandos en MAYÚSCULAS para distinguirlos de los nombres de tus tablas. 

SELECT columna FROM tabla;
El punto y coma `;` final es obligatorio en la mayoría de sistemas.

3. SELECT & FROM

Para ver datos. 

-- Trae TODO (*) de la tabla usuarios
SELECT * FROM usuarios;

-- Trae solo nombres y correos
SELECT nombre, email FROM usuarios;
Tip: Evita usar `SELECT *` en aplicaciones reales si la tabla es gigante; trae solo lo que necesites.

4. La Cláusula WHERE

Para filtrar. Sin ella, traerías millones de registros. 

SELECT * FROM usuarios
WHERE pais = 'Mexico' AND edad > 18;
Operadores: `=`, `>`, `<`, `<>` (diferente), `LIKE` (para buscar texto similar).

5. INSERT, UPDATE, DELETE (CRUD)

Modificar datos es delicado. 

-- Insertar
INSERT INTO usuarios (nombre, edad) VALUES ('Ana', 25);

-- Actualizar (¡Cuidado con el WHERE!)
UPDATE usuarios SET edad = 26 WHERE nombre = 'Ana';

-- Borrar (¡PELIGRO!)
DELETE FROM usuarios WHERE id = 5;

Evaluación 2: Seguridad

¿Qué sucede si ejecutas DELETE FROM usuarios; sin un WHERE?

Borras TODA la tabla (Pánico).
Solo borra el primer usuario.

6. JOINS (Las uniones)

El poder real de SQL: Cruzar datos de tablas diferentes. 

SELECT ordenes.id, usuarios.nombre
FROM ordenes
INNER JOIN usuarios ON ordenes.usuario_id = usuarios.id;
Conecta las órdenes con los nombres de quienes las compraron.

7. Agrupaciones (GROUP BY)

Para reportes y estadísticas. 

-- Contar usuarios por país
SELECT pais, COUNT(*)
FROM usuarios
GROUP BY pais;
Funciones de agregación: `COUNT`, `SUM`, `AVG` (promedio), `MAX`, `MIN`.

8. Llaves (Keys)

  • Primary Key (PK): ID único (ej. Cédula, ID). No se repite nunca.
  • Foreign Key (FK): Referencia a una PK de otra tabla. Crea la relación.

9. Normalización

Es ordenar tu casa.
Reglas para evitar datos duplicados. Si guardas la dirección del usuario 50 veces en 50 órdenes, está mal. Mejor guarda la dirección UNA vez en la tabla usuarios y relaciónala.

10. Índices

Hacen que las búsquedas sean instantáneas.
Sin índice, la DB lee fila por fila (Full Table Scan). Con índice, va directo al grano.
Coste: Ocupan espacio y hacen que guardar datos nuevos sea un poco más lento (porque hay que actualizar el índice).

11. NoSQL vs SQL

SQL (Estructurado): Bancos, contabilidad, datos críticos y relacionados.
NoSQL (Flexible): Redes sociales, Big Data, prototipos rápidos. No usa tablas fijas.
