1. ¿Qué es una Base de Datos?

Es una colección organizada de información. Las bases de datos relacionales (como MySQL, PostgreSQL) guardan datos en tablas filas y columnas.

2. Sintaxis Básica

SQL (Structured Query Language) es el idioma estándar. Se escribe en mayúsculas por convención, aunque no es obligatorio.
`SELECT * FROM usuarios;`

3. SELECT & FROM

El "Hola Mundo" de SQL.
`SELECT nombre, edad FROM usuarios;`
Trae solo las columnas 'nombre' y 'edad' de la tabla 'usuarios'.

4. La Cláusula WHERE

Filtra los resultados.
`SELECT * FROM productos WHERE precio > 100;`

5. INSERT, UPDATE, DELETE

Manipular datos (CRUD):
`INSERT INTO ... VALUES ...`
`UPDATE ... SET ...`
`DELETE FROM ...`

Evaluación 2: Peligro

¿Qué pasa si ejecutas un DELETE sin WHERE?

Borras TODOS los registros de la tabla.
Te da un error de sintaxis.
Solo borra el último.

6. JOINS

Combina filas de dos o más tablas.
`INNER JOIN`: Solo coincidencias.
`LEFT JOIN`: Todo lo de la izquierda.

7. GROUP BY & ORDER BY

`GROUP BY`: Agrupa filas idénticas (para sumas, promedios).
`ORDER BY`: Ordena ascendente (ASC) o descendente (DESC).

8. Llaves (Keys)

**Primary Key**: Identificador único de cada fila (ID).
**Foreign Key**: Columna que apunta a la Primary Key de otra tabla.

Evaluación 3: Relaciones

¿Una Foreign Key puede repetirse?

Sí, (Relación 1 a Muchos).
No, nunca.

9. Normalización

Proceso de organizar datos para reducir redundancia. 1NF, 2NF, 3NF. Busca evitar duplicados.

10. Índices

Estructuras de datos especiales para acelerar las consultas. Como el índice de un libro. Hacen los SELECT rápidos, pero los INSERT lentos.

11. NoSQL vs SQL

NoSQL (MongoDB, Firebase) no usa tablas, sino documentos JSON. Es más flexible pero menos estricto con las relaciones.
