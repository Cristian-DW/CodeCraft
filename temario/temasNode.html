Volver al Curso

1. ¿Qué es Node.js?

No es un lenguaje. Es un entorno de ejecución de JavaScript construido sobre el motor V8 de Chrome. Permite crear servidores, herramientas de línea de comandos y más.

2. Instalación

Se descarga desde nodejs.org. Viene con NPM (Node Package Manager).
Comando para verificar: `node -v`.

Evaluación 1: Conceptos

¿Node.js es un framework frontend?

Sí, reemplaza a React.
No, es un entorno de ejecución backend.
Es una base de datos.

3. Módulos

Node usa módulos para organizar código.
CommonJS: `const fs = require('fs');`
ES Modules (Moderno): `import fs from 'fs';`

4. File System (fs)

Módulo nativo para leer y escribir archivos en el servidor.
`fs.readFile('archivo.txt', ...)`

5. Event Loop

El secreto de la velocidad de Node. Es "Non-blocking I/O". Puede manejar miles de conexiones concurrentes sin crear nuevos hilos.

Evaluación 2: Arquitectura

¿Node.js es bloqueante o no bloqueante?

Bloqueante (Espera a que termine una tarea)
No Bloqueante (Asíncrono)

6. Módulo HTTP

Permite crear un servidor web básico.
`http.createServer((req, res) => ...)`

7. NPM y package.json

NPM es el registro de software más grande del mundo.
`package.json` es el archivo donde configuras tu proyecto y sus dependencias.

8. Express.js

Framework minimalista para Node. Simplifica la creación de servidores.
`const app = express();`

Evaluación 3: NPM

¿Qué comando instala una librería nueva?

npm install nombre-paquete
node add nombre-paquete
git clone nombre-paquete

9. Rutas (Routing)

Cómo responde tu aplicación a diferentes URLs.
`app.get('/', (req, res) => res.send('Hola'))`

10. Middleware

Funciones que se ejecutan entre la petición y la respuesta. Se usan para autenticación, logs, etc.

11. Bases de Datos

Node conecta bien con todo.
SQL: MySQL, PostgreSQL.
NoSQL: MongoDB (muy popular con Node, stack MERN).
