1. ¿Qué es Node.js?

Muchos creen que es un lenguaje, pero NO lo es.

Node.js es un entorno de ejecución para JavaScript construido sobre el motor V8 de Chrome (el mismo que usa tu navegador).
Te permite correr JS fuera del navegador: en servidores, en tu computadora, en robots, etc.

2. Instalación y NPM

Al instalar Node.js, automáticamente instalas NPM (Node Package Manager). 

$ node -v
v18.16.0

$ npm -v
9.5.1
NPM es la tienda de aplicaciones más grande para desarrolladores. ¿Necesitas crear PDFs? Hay un paquete. ¿Manejar fechas? Hay un paquete.

3. Sistema de Módulos

Node organiza el código en piezas reutilizables. 

// Forma antigua (CommonJS)
const fs = require('fs');

// Forma moderna (ES Modules)
import fs from 'fs';
Esto permite tener archivos pequeños y mantenibles en lugar de un archivo de 10,000 líneas.

4. File System (fs)

A diferencia del navegador, Node.js tiene acceso a tus archivos. 

import fs from 'fs';

// Leer un archivo de texto
const contenido = fs.readFileSync('nota.txt', 'utf-8');
console.log(contenido);
Poderoso, pero peligroso si no tienes cuidado con la seguridad.

5. Event Loop (El Camarero)

Node.js es Asíncrono y No Bloqueante.

Imagine un camarero (Node):

  • Toma tu pedido y lo pasa a la cocina.
  • Inmediatamente va a atender a otra mesa (no espera en la cocina).
  • Cuando la cocina termina, el camarero te trae la comida.
Esto le permite manejar miles de conexiones a la vez con un solo hilo.

Evaluación 2: Arquitectura

¿Qué pasa si una tarea tarda mucho en Node.js?

Se congela todo el servidor (Bloqueo).
Node sigue atendiendo otras peticiones (No Bloqueo).

6. Servidor HTTP Nativo

Puedes crear un servidor sin instalar nada extra: 

import http from 'http';

const server = http.createServer((req, res) => {
    res.write('¡Hola Mundo desde Node!');
    res.end();
});

server.listen(3000);
Aunque es posible, es muy verboso. Por eso usamos frameworks.

7. Express.js

El framework estándar. Simplifica todo. 

import express from 'express';
const app = express();

app.get('/', (req, res) => {
    res.send('¡Hola Express!');
});

app.listen(3000);
Menos código, más fácil de leer.

8. Rutas (Routing)

Responder diferente según la URL. 

app.get('/usuarios', (req, res) => { ... });
app.post('/login', (req, res) => { ... });
app.delete('/borrar', (req, res) => { ... });
Cada "verbo" HTTP (GET, POST, DELETE) tiene su función.

9. Middleware

Funciones intermedias. Imagina un guardia de seguridad antes de entrar al club. 

app.use((req, res, next) => {
    console.log("Alguien entró...");
    next(); // Deja pasar al siguiente paso
});
Se usan para verificar logins, leer cookies o procesar datos.

10. Bases de Datos

Node.js es el mejor amigo de MongoDB (Base de datos NoSQL).

Juntos forman el stack MERN (Mongo, Express, React, Node). Mongo guarda los datos como documentos JSON, lo que es muy natural para JavaScript.

11. Despliegue (Deploy)

Tu servidor Node no vive en tu laptop.
Servicios como Render, Railway o AWS permiten subir tu código para que funcione 24/7 en la nube y sea accesible desde cualquier lugar.
