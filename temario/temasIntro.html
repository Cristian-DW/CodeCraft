1. Historia de la Programación

La programación no nació con las computadoras modernas.

Hitos importantes:

  • Ada Lovelace (1843): Escribió el primer algoritmo para la Máquina Analítica de Babbage.
  • Grace Hopper (1950s): Inventó el primer compilador y popularizó el término "bug" (insecto) cuando una polilla causó un fallo en una computadora Mark II.
  • La Era Moderna: Desde C y Unix en los 70s hasta la Web (HTML/HTTP) creada por Tim Berners-Lee en 1989.
Programar es, en esencia, dar instrucciones precisas a una máquina para resolver problemas humanos.

2. Lógica y Algoritmos

Un algoritmo es como una receta de cocina: una serie de pasos ordenados y finitos.

Ejemplo: Algoritmo para lavarse los dientes

  1. Tomar el cepillo y la pasta.
  2. Poner pasta en el cepillo.
  3. Cepillar cada diente por 2 minutos.
  4. Enjuagar.
Si te saltas un paso o cambias el orden (enjuagar antes de cepillar), el resultado falla. Eso es lógica.

Evaluación 1: Lógica

¿Qué característica NO pertenece a un algoritmo?

Debe tener un orden específico.
Debe ser infinito y nunca terminar.
Debe tener un resultado o fin.

3. Tipos de Datos

Así como en la vida real distingues entre letras y números, la computadora también lo hace.

  • String (Texto): Una cadena de caracteres. Siempre va entre comillas. 
    "Hola Mundo"
  • Integer (Entero): Números sin decimales. 
    42, -10, 2026
  • Float (Flotante): Números con decimales. 
    3.1416, 9.99
  • Boolean (Lógico): Solo tiene dos valores posibles. 
    true (verdadero), false (falso)

4. Variables y Constantes

La Analogía de la Caja:
Imagina que tienes una caja etiquetada como "Edad".

- Variable: Puedes meter el número 20 hoy, y cambiarlo por 21 el próximo año. El contenido varía.
- Constante: Es una caja sellada con pegamento industrial. Una vez metes el valor (ej. tu fecha de nacimiento), nunca puedes cambiarlo.

En código, usamos variables para guardar datos que usaremos después.

5. Tu entorno de trabajo (IDE)

Para programar no usas Word, usas un Editor de Código.

Visual Studio Code (VS Code) es el rey actual. ¿Por qué?

  • Resalta la sintaxis con colores.
  • Te autocompleta el código (IntelliSense).
  • Tiene miles de Extensiones (como Live Server para ver tu web en tiempo real).

6. ¿Cómo funciona Internet?

Internet son cables submarinos conectando computadoras.

El proceso simplificado:

  1. Tú escribes google.com.
  2. Tu navegador pregunta a un DNS (la agenda telefónica de internet) cuál es la IP de Google.
  3. El DNS responde: 142.250.189.46.
  4. Tu navegador va a esa dirección IP y pide la página web.

7. Cliente vs Servidor

Cliente (Tú/Front-end): Pide recursos (imágenes, webs). Es tu navegador o celular.
Servidor (Ellos/Back-end): Una computadora poderosa encendida 24/7 que tiene los archivos y te los envía cuando los pides.

8. Frontend vs Backend

Frontend (La Cara)
Lo que el usuario ve y toca.
Tecnologías: HTML, CSS, JavaScript, React.
Backend (El Cerebro)
Lógica, datos y seguridad.
Tecnologías: Node.js, Python, SQL, Java.

9. Sistemas Operativos

Aunque uses Windows o macOS en casa, Linux domina el mundo de los servidores e internet.

¿Por qué? Porque es gratuito, de código abierto, muy seguro y eficiente. Como desarrollador, tarde o temprano tendrás que usar Linux para desplegar tus aplicaciones.

10. La Terminal
