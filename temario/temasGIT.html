Volver al Curso

1. ¿Qué es Control de Versiones?

Es un sistema que registra los cambios en un archivo o conjunto de archivos a lo largo del tiempo. Te permite volver a versiones anteriores (como un "Ctrl+Z" gigante).

2. Instalación

Git es software libre. En Mac suele venir instalado. En Windows se descarga Git Bash. Configura tu identidad:
`git config --global user.name "Tu Nombre"`

Evaluación 1: Conceptos

¿Git y GitHub son lo mismo?

Sí, son sinónimos.
No. Git es el software, GitHub es la plataforma web.

3. Init & Clone

`git init`: Crea un repositorio nuevo en tu carpeta.
`git clone [url]`: Descarga un repositorio existente desde internet.

4. Add & Commit

El ciclo de vida:
1. Haces cambios.
2. `git add .` (Mueve a Staging Area).
3. `git commit -m "mensaje"` (Guarda la foto).

5. Status & Log

`git status`: ¿Qué ha cambiado? ¿Qué está en verde/rojo?
`git log`: Historial de todos los commits hechos.

Evaluación 2: Comandos

¿Qué comando guarda los cambios permanentemente en el historial?

git save
git add
git commit

6. Ramas (Branches)

Universos paralelos. `git branch nueva-feature` crea una copia donde puedes romper cosas sin afectar la rama principal (main).

7. Fusiones (Merge)

Unir una rama con otra. `git merge nueva-feature` trae los cambios de vuelta a main.

8. Conflictos

Ocurren cuando dos personas editan la misma línea. Git te pide que elijas qué código quedarte. Se resuelven manualmente.

Evaluación 3: Ramas

¿Cuál es la rama principal por defecto?

main (antes master)
trunk
index

9. GitHub / GitLab

Servicios en la nube para alojar tus repositorios Git. Permiten colaboración, backups y CI/CD.

10. Pull Requests

La forma profesional de hacer merge. Pides permiso para unir tus cambios. Tus compañeros revisan el código antes de aceptar.

11. .gitignore

Un archivo de texto donde listas carpetas y archivos que Git debe ignorar (ej. node_modules, contraseñas).

Evaluación Final

¿Qué comando descarga cambios remotos a tu local?

git pull
git push
git download
