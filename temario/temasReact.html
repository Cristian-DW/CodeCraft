Volver al Curso

1. ¿Qué es React?

Librería de JavaScript para construir interfaces de usuario. Basada en componentes. Creada por Facebook.

2. JSX (JavaScript XML)

Parece HTML, pero es JS. Permite escribir etiquetas dentro del código JavaScript.
`const elemento = <h1>Hola</h1>;`

Evaluación 1: Sintaxis

¿Puedes escribir HTML puro en un archivo .js sin transpilar?

Sí, funciona siempre.
No, requiere JSX y un transpilador (Babel).
Solo si usas jQuery.

3. Componentes

Piezas de UI reutilizables. Pueden ser funciones o clases (hoy en día usamos funciones).
`function Boton() { return <button>Click</button>; }`

4. Props (Propiedades)

Información que pasas de un componente padre a un hijo. Son de solo lectura.
`<Saludo nombre="Juan" />`

5. Estado (State)

La memoria del componente. Si cambia el estado, el componente se vuelve a renderizar para mostrar los cambios.

Evaluación 2: Hooks

¿Qué Hook se usa para manejar el estado en componentes funcionales?

useEffect
useState
useHistory

6. Manejo de Eventos

Son como los de HTML pero en camelCase.
`onclick` -> `onClick`.
`<button onClick={manejarClick}>`

7. Renderizado Condicional

Mostrar u ocultar cosas según condiciones.
`{ logueado ? <Perfil /> : <Login /> }`

8. Listas y Keys

Usamos `.map()` para convertir arrays de datos en arrays de elementos. Siempre necesitas una `key` única.

Evaluación 3: Listas

¿Por qué React necesita una "key" en las listas?

Para identificar qué elementos han cambiado.
Para dar estilos CSS automáticamente.
Es opcional.

9. useEffect

Para efectos secundarios: llamadas a APIs, suscripciones, timers. Reemplaza a `componentDidMount` y `componentDidUpdate`.

10. Formularios Controlados

En React, el estado es la única fuente de la verdad. Los inputs leen del estado y escriben al estado (`onChange`).

11. Build & Deploy

`npm run build`. Crea una versión optimizada y minificada de tu app lista para subir a producción (Vercel, Netlify).

Evaluación Final

¿React manipula el DOM real directamente todo el tiempo?

Sí, por eso es lento.
No, usa un Virtual DOM para optimizar cambios.
