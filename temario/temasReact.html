Volver al Curso

1. ¿Qué es React?

Es una librería creada por Facebook (Meta) para construir interfaces.

Filosofía: "Divide y vencerás". En lugar de una página gigante, construyes pequeños Componentes (Botón, Header, Tarjeta) que unes como piezas de LEGO.

2. JSX (JavaScript XML)

React usa una sintaxis mágica que mezcla JS con HTML. 

const nombre = "Cristian";
const elemento = <h1>Hola, {nombre}!</h1>;
Todo lo que va entre llaves `{}` se evalúa como JavaScript. ¡Es muy poderoso!

Evaluación 1: Conceptos

¿JSX es HTML estándar?

Sí, el navegador lo entiende nativamente.
No, es una extensión de JavaScript que necesita ser compilada.

3. Componentes

Un componente es simplemente una función que devuelve UI. 

function Boton() {
    return <button>Click aquí</button>;
}

// Uso
<Boton />
Deben empezar siempre con Mayúscula.

4. Props (Propiedades)

Así es como los componentes se comunican. El padre le pasa datos al hijo. 

function Saludo(props) {
    return <h2>Hola, {props.nombre}</h2>;
}

<Saludo nombre="Ana" /> // Renderiza: Hola, Ana
Las props son INMUTABLES (no puedes cambiarlas, solo leerlas).

5. Estado (useState)

Si las props son datos externos, el Estado son datos internos del componente. 

import { useState } from 'react';

function Contador() {
    const [cuenta, setCuenta] = useState(0);

    return <button onClick={() => setCuenta(cuenta + 1)}>
        Clicks: {cuenta}
    </button>;
}
Cuando el estado cambia, React actualiza la pantalla automáticamente.

Evaluación 2: Hooks

¿Qué función usamos para modificar el estado?

cuenta = 5;
setCuenta(5);
update(cuenta, 5);

6. Renderizado Condicional

Mostrar cosas diferentes según el estado. 

function Panel({ usuario }) {
    return (
        <div>
            {usuario ? <Bienvenida /> : <LoginButton />}
        </div>
    );
}
Usamos el operador ternario `condición ? verdadero : falso` constantemente.

7. Listas y Keys

Transformamos arrays de datos en arrays de componentes con `.map()`. 

const frutas = ['Manzana', 'Pera'];

return (
    <ul>
        {frutas.map(fruta => <li key={fruta}>{fruta}</li>)}
    </ul>
);
La key es obligatoria para que React sepa qué ítem cambió sin redibujar toda la lista.

8. Eventos

React usa camelCase para eventos: `onClick`, `onSubmit`.
Además, pasamos la función, NO la ejecutamos.

Correcto: `onClick={hacerAlgo}`
Incorrecto: `onClick={hacerAlgo()}` (esto la ejecutaría al cargar).

Evaluación 3: Sintaxis

¿Cuál es la forma correcta de asignar una clase CSS en React?

class="boton"
className="boton"

9. useEffect (Efectos)

Para cosas que pasan "fuera" del renderizado: pedir datos a una API, poner un timer, etc. 

useEffect(() => {
    console.log("El componente se montó");
}, []); // Array vacío = Ejecutar solo al inicio

10. El Virtual DOM

React es rápido porque tiene un "borrador" de tu página en memoria. Cuando algo cambia, compara el borrador con la página real y actualiza solo lo que cambió.
Esto es mucho más eficiente que recargar todo.

11. El Ecosistema

React es solo la vista. Para una app completa necesitas:

  • Routing: React Router.
  • Estilos: Styled Components, TailwindCSS.
  • Estado Global: Redux, Zustand, Context API.
