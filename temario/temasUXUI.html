1. UX vs UI

UX (User Experience): Cómo se siente usar el producto (fácil, útil).
UI (User Interface): Cómo se ve el producto (colores, botones).

2. Design Thinking

Metodología de 5 pasos: Empatizar, Definir, Idear, Prototipar y Testear. Poner al usuario en el centro.

Evaluación 1: Conceptos

¿Un botón bonito que no funciona es un fallo de UX o UI?

Fallo de UI (Visual)
Fallo de UX (Experiencia/Funcionalidad)

3. Teoría del Color

60-30-10. Usa un color dominante (60%), uno secundario (30%) y uno de acento (10%). Entiende el contraste y la psicología.

4. Tipografía

No uses más de 2 fuentes. Usa **Sans-Serif** (sin remates) para pantallas por su legibilidad (como Inter o Roboto).

5. Espaciado y Grillas

El espacio en blanco es tu amigo. Da respiro al diseño. Usa grillas de 8pt o 12 columnas para alinear todo perfectamente.

Evaluación 2: Visual

¿Qué significa 'Whitespace'?

Espacio vacío entre elementos (no necesariamente blanco).
Texto de color blanco.

6. Wireframing

Bocetos de baja fidelidad (Low-Fi). Estructura básica sin colores ni detalles. Como el plano de una casa.

7. Prototipado (Figma)

Crear versiones interactivas de tu diseño. Figma es la herramienta líder del mercado.

8. Pruebas con Usuarios

Observar a gente real usando tu app. Descubrirás que lo que para ti es obvio, para ellos no lo es.

Evaluación 3: Herramientas

¿Cuál es la herramienta estándar hoy en día para UI?

Figma
Paint
Photoshop (para UI web)

9. Accesibilidad (a11y)

Diseñar para todos, incluyendo personas con discapacidad visual o motora. Usa buen contraste y etiquetas alt.

10. Mobile First

Diseña primero para la pantalla pequeña del celular, luego escala a escritorio. Obliga a priorizar contenido.

11. Sistemas de Diseño

Una biblioteca de componentes (botones, inputs, colores) reutilizables para mantener consistencia en todo el producto.

Evaluación Final

Sí, al nivel de un ingeniero.
