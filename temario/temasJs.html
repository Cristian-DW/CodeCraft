1. ¿Qué es JavaScript?

Si HTML es el esqueleto y CSS la piel, JavaScript (JS) son los músculos y el sistema nervioso.

Es un lenguaje de programación real (a diferencia de HTML/CSS) que permite lógica: "Si el usuario hace click aquí, muestra esto".

Se puede ejecutar en el navegador (Chrome, Firefox) o en el servidor (Node.js).

2. Variables Modernas (ES6)

En el JS moderno (2015+), evitamos `var`. 

// let: Cajita abierta. Puedes cambiar el valor.
let vidas = 3;
vidas = 2; // Ok

// const: Cajita sellada. NO puedes cambiar el valor.
const PI = 3.1416;
PI = 4; // Error!
Regla de oro: Usa `const` por defecto. Si necesitas reasignar, usa `let`. Nunca uses `var`.

Evaluación 1: Conceptos Básicos

¿Qué pasará si intento ejecutar: `const nombre = "Juan"; nombre = "Pedro";`?

La variable cambia a "Pedro".
El navegador lanza un Error (Assignment to constant variable).
El navegador se cierra.

3. Tipos de Datos

Datos Primitivos vs Objetos.

  • String: `"Hola"` o `'Hola'`.
  • Number: `10` o `10.5` (todos son flotantes internamente).
  • Boolean: `true` / `false`.
  • Undefined: Variable creada pero sin valor.
  • Null: Ausencia intencional de valor.

4. Operadores

¡Cuidado con las comparaciones! 

5 == "5"  // true (convierte el tipo, peligroso)
5 === "5" // false (estricto, verifica tipo y valor)
Usa siempre la Triple Igualdad (`===`) para evitar errores extraños.

5. Condicionales

Controlan el flujo del programa. 

const edad = 18;

if (edad >= 18) {
    console.log("Puedes votar");
} else {
    console.log("Eres menor");
}
La computadora lee línea por línea, pero con `if` puede saltarse bloques enteros.

6. Bucles (Loops)

Para no repetir código manualmente. 

// Repetir 5 veces
for (let i = 0; i < 5; i++) {
    console.log("Vuelta " + i);
}
Ideal para recorrer listas de datos (ej. lista de productos).

7. Funciones Flecha (Arrow Functions)

Las funciones son bloques de código reutilizables. 

// Antigua
function sumar(a, b) { return a + b; }

// Moderna (Arrow)
const sumar = (a, b) => a + b;
Más limpias y concisas.

8. El DOM

El DOM (Document Object Model) es el puente entre JS y HTML. 

// Seleccionar el título h1
const titulo = document.querySelector('h1');

// Cambiar su texto
titulo.textContent = "¡Nuevo Título!";

// Cambiar su color
titulo.style.color = "red";

9. Eventos

Hacer que la web reaccione al usuario. 

const boton = document.querySelector('.btn');

boton.addEventListener('click', () => {
    alert('¡Hiciste click!');
});
Eventos comunes: `click`, `submit` (formularios), `input` (escribir), `scroll`.

Evaluación 3: Interacción

'click'

10. Arrays y Objetos

Estructuras de datos esenciales. 

// Array (Lista)
const frutas = ["Manzana", "Banana"];
console.log(frutas[0]); // Manzana

// Objeto (Diccionario)
const usuario = { nombre: "Ana", edad: 25 };
console.log(usuario.nombre); // Ana

11. Fetch API (Async)

Cómo JS pide datos a servidores externos (APIs) sin recargar la página. 

fetch('https://api.clima.com')
  .then(respuesta => respuesta.json())
  .then(datos => console.log(datos));
Es la base de las aplicaciones modernas como Instagram o Twitter.

Evaluación Final

¿Qué significa que JavaScript es un lenguaje "Interpretado"?

Que necesita ser compilado a binario antes de correr.
Que el navegador lee y ejecuta el código línea por línea al momento.
